**Key finding — Visa ref format split:**
The CSV has two memo formats. Before 2026-01-22, memos are short 4-digit
codes that can collide (proven: memo `2322` shared by Walmart and DoorDash).
After 2026-01-22, memos are semicolon-delimited with long native refs
(e.g., `24445006022000789209313`).

Decision: short memos get composite refs (`date|memo|amount`), long memos
get native refs (first token). Native refs auto-mark as duplicate during
dedup; composite refs always flag for review. This is consistent with the
pipeline plan's native vs composite distinction.

**Promote JSON parsing bug:** The CLI outputs log lines before the JSON
block. The original parser looked for the last line; the fix looks for the
first `{` and parses from there. The 364 JE IDs were backfilled via a SQL
UPDATE matching on reference values. The fix is in the script for next time.
