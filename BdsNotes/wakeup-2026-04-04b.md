# Wakeup — 2026-04-04b (Project 030 Ready to Build)

## What happened this session

### Brainstormed and planned Project 030: Unfuck Our Test Harness

Full design conversation with Dan about replacing TickSpec with plain xUnit.
Started from the connection leak diagnosis in wakeup-2026-04-04a, traced
the root causes, and designed the new architecture through collaborative
brainstorming.

Key decisions made during brainstorm:
- Gherkin stays as spec (source of truth), not test framework
- NpgsqlDataSource singleton in LeoBloom.Dal — eager init, private internals,
  one public `openConnection()` function
- Tests call real production functions — no test-only `*InTransaction` variants
- Commit + delete cleanup (not transaction rollback)
- One test project (LeoBloom.Tests), one pattern for everything
- Designed for parallelism with unique test data

### Deepened the plan with 7 review/research agents

Architecture strategist, code simplicity reviewer, pattern recognition
specialist, performance oracle, data integrity guardian, best practices
researcher, framework docs researcher.

Key improvements from deepening:
- **Eager init, not lazy** — avoids `Lazy<T>` cached exception problem where
  a failed init poisons every subsequent call forever
- **TestCleanup as F# module** (not OOP class) — matches codebase conventions
- **Cleanup failures logged** to stderr, not silently swallowed
- **DataSource config**: `IncludeErrorDetail=true`, `ApplicationName="LeoBloom"`,
  `Timeout=5`, `CommandTimeout=5`
- **Prod safety guard** integrated into eager init for free
- **Phase 3 executes before Phase 2** — consolidate test projects before
  refactoring production code, so Domain tests stay green throughout
- **Parallel isolation constraint** documented — always query by specific IDs
