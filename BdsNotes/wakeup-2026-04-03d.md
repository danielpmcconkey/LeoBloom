### Design decisions made this session

1. **DUs are source of truth for enumerated values.** DB stores human-readable strings, not int FKs to lookup tables. Executive decision from Dan.
2. **account_type stays as a table.** It carries `normal_balance` — that's a real attribute serving 3NF, not a pure lookup. `account`, `fiscal_period` also stay. Only pure name-only lookups get eliminated.
3. **No CHECK constraints.** Dan considers them as stupid as stored procedures for CRUD. Don't bring it up.
4. **Whitespace void reasons are a write-level concern.** The domain function handles it (IsNullOrWhiteSpace), but testing it belongs at the service/persistence layer (backlog item 028), not as an isolated domain unit test.
