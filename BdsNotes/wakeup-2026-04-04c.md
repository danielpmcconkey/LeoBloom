### Found and fixed a real bug

AccountBalanceRepository.getBalance had a LEFT JOIN that didn't exclude
voided entries from balance calculations. The `LEFT JOIN journal_entry je
ON ... AND je.voided_at IS NULL` pattern returns NULL for je columns but
keeps jel (line) rows — so voided entry amounts were included in sums.
Changed to nested JOIN. Old TickSpec tests couldn't catch this because
they ran inside transactions where void and query shared context.

### Fixed cleanup data leaks

TestCleanup.deleteAll originally only deleted transfers/instances by
journal_entry_id. Tests that inserted transfers or obligation_instances
without journal entries left orphaned rows that blocked account/type
cleanup via FK. Fixed by adding multi-column delete paths: transfers by
account_id, instances by agreement_id, agreements by account_id.

### Safety guard → #if DEBUG

Dan ran tests from Rider on the host. LEOBLOOM_ENV=Production on the host
loaded prod config, but pg_hba.conf rejected the connection (host IP not
allowed) — no data touched. But it exposed that the hardcoded
`current_database() = leobloom_dev` guard would break prod usage entirely.
Wrapped in `#if DEBUG` so it compiles out of release builds.
