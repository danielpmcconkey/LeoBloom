# Wakeup — 2026-04-02a (Leo Bloom — Project 1 Orientation)

## What is Leo Bloom?

Custom full-stack financial management system. Double-entry bookkeeping (cash
basis, GAAP-informed), obligation tracking, nagging agent, invoice generation,
balance projection, UI. Named after Gene Wilder's accountant in The Producers.

This is a hobby project. Dan is learning GAAP by modelling his own finances.
"FUN in the Dwarf Fortress sense."

## Tech Stack

- **F# all the way down** — domain, API (Giraffe/Falco), data access, frontend
  (Fable/Feliz/Elmish), AND the Migrations console app
- **PostgreSQL** — two databases on the host Postgres instance:
  - `leobloom_dev` — your database. Role: `claude`. Already created.
  - `leobloom_prod` — Hobson/Dan's database. You don't touch this.
- **Schema-first migrations** — SQL migration files are source of truth
- **.NET 10** — install it if you don't have it. The host has 10.0.201.
- **.sln at repo root**

## What to read (in this order)

1. `Specs/DataModelSpec.md` — the data model. Ledger + ops schemas.
2. `Projects/Project1-Database.md` — the BRD for what you're building. Schema,
   migrations, seed data, Gherkin acceptance specs for structural constraints.
3. `Specs/SampleCOA.md` — anonymized chart of accounts for dev seed data.
